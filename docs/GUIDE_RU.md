# Инструкция пользователя: Компонент WebpConverter для MODX 3

Компонент **WebpConverter** позволяет автоматически ускорить загрузку вашего сайта на MODX 3 за счет конвертации изображений (JPEG, PNG) в современный легковесный формат **WebP**. Плагин автоматически заменяет ссылки на картинки на лету, отдавая пользователям оптимизированные версии.

---

## 1. Установка компонента

1. Перейдите в панель управления MODX (админку).
2. Откройте меню **Приложения** (Extras) -> **Управление пакетами** (Installer).
3. Нажмите кнопку **Загрузить пакеты** и выберите транспортный пакет `webpconverter-1.0.0-pl.transport.zip`.
4. Нажмите **Установить** рядом с загруженным пакетом и следуйте инструкциям на экране.

После установки в системе появятся следующие элементы:
- Системный плагин, который на лету заменяет ссылки на изображения на `.webp`.
- Системное меню **WEBP Конвертер** в разделе **Приложения** (Extras).
- Виджет **WEBP Оптимизация** на главной странице панели управления (Dashboard).

## 2. Системные настройки компонента

Настройки компонента находятся в разделе **Системные настройки** (шестеренка в верхнем правом углу -> Системные настройки), в пространстве имен `webpconverter`.

### Доступные параметры:

1. **`webpconverter.cwebp_params_jpeg`** (Параметры сжатия JPEG)
   - **Значение по умолчанию**: `-metadata none -quiet -pass 10 -m 6 -mt -q 65 -low_memory`
   - **Описание**: Параметры командной строки для утилиты `cwebp`, используемые при сжатии JPEG.
     - `-metadata none` — удаляет все метаданные EXIF/IPTC для уменьшения размера файла.
     - `-quiet` — отключает подробный вывод утилиты в лог.
     - `-pass 10` — количество проходов для оптимизации распределения битрейта (максимум 10).
     - `-m 6` — метод сжатия (0-6, где 6 обеспечивает наилучшее сжатие, но требует больше процессорного времени).
     - `-mt` — включение многопоточности (Multi-threading) для ускорения обработки.
     - `-q 65` — качество сжатия (шкала 0-100, где 65 — оптимальный баланс качества и веса для JPEG).
     - `-low_memory` — оптимизация использования оперативной памяти сервером.

2. **`webpconverter.cwebp_params_png`** (Параметры сжатия PNG)
   - **Значение по умолчанию**: `-metadata none -quiet -pass 10 -m 6 -alpha_q 85 -mt -alpha_filter best -alpha_method 1 -q 70 -low_memory`
   - **Описание**: Параметры командной строки для сжатия PNG с поддержкой прозрачности.
     - `-alpha_q 85` — качество сжатия альфа-канала (прозрачности).
     - `-alpha_filter best` — алгоритм фильтрации альфа-канала.
     - `-alpha_method 1` — метод сжатия прозрачности.
     - `-q 70` — качество сжатия основных цветов PNG.

3. **`webpconverter.exclude_dirs`** (Исключаемые директории)
   - **Значение по умолчанию**: `core,connectors,manager,webp,tmp,.git,vendor,node_modules`
   - **Описание**: Список папок через запятую (относительно корня сайта), которые сканер картинок должен полностью игнорировать при поиске. Рекомендуется исключать системные папки MODX и служебные директории проектов, чтобы избежать лишней нагрузки на сервер.

4. **`webpconverter.disable_for_logged_user`** (Отключить для авторизованных пользователей)
   - **Значение по умолчанию**: `Нет` (false)
   - **Описание**: Если включить эту настройку (`Да` / true), плагин не будет подменять пути к изображениям на `.webp` для администраторов и менеджеров, авторизованных в панели управления MODX. Это полезно при тестировании оригинального отображения или отладке верстки.

## 3. Использование компонента

### Главная панель управления (Интерфейс приложения)

Перейдите в меню **Приложения** -> **WEBP Конвертер**. Перед вами откроется рабочая панель:

1. **Сканирование сайта**:
   - Нажмите кнопку **Сканировать сайт**, чтобы запустить поиск всех изображений на сервере. Сканер найдет все файлы форматов JPG, JPEG и PNG, кроме расположенных в исключенных папках.
   - Во время сканирования отображается живой индикатор прогресса.
2. **Конвертация изображений**:
   - Нажмите кнопку **Конвертировать файлы**, чтобы запустить процесс генерации `.webp` версий.
   - Конвертация происходит пакетами (по умолчанию по 10 файлов за один запрос) для предотвращения тайм-аутов сервера. Индикатор будет наглядно показывать процент завершения и статус выполнения.
3. **Очистка лишних файлов**:
   - Кнопка **Очистить лишние файлы** позволяет удалить с сервера файлы `.webp`, оригиналы которых были удалены (например, если вы удалили картинку из галереи или файлового менеджера). Это помогает освободить дисковое пространство.

### Виджет на главной странице (Dashboard Widget)

На главной странице админки отображается удобный виджет **WEBP Оптимизация**, в котором выводится актуальная статистика:
- Общее количество изображений на сайте.
- Количество уже оптимизированных изображений.
- Текущий прогресс оптимизации в процентах.
- Общий сэкономленный дисковый объем (в мегабайтах или килобайтах).
- Кнопка **Обновить данные** для мгновенного пересчета статистики.
- Кнопка **Перейти в конвертер** для быстрого перехода к ручному управлению.
