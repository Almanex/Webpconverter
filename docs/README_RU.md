WebpConverter — это готовое решение для MODX 3, которое автоматически ускоряет загрузку страниц сайта за счет конвертации изображений JPEG и PNG в современный формат WebP. Компонент на лету подменяет пути к картинкам в HTML-коде и предоставляет панель управления со статистикой сжатия.

---

## Ключевые возможности

- **Динамическая замена путей**: Автоматическая подмена ссылок в тегах `<img>`, списках `srcset` и инлайн-стилях на `.webp` версии при выводе страницы.
- **Сканирование сайта**: Рекурсивный поиск файлов в директориях сервера с поддержкой настраиваемого списка исключений.
- **Пакетная конвертация**: Генерация WebP-файлов небольшими порциями для предотвращения таймаутов сервера и перегрузки процессора.
- **Очистка сиротских файлов**: Автоматическое удаление файлов `.webp`, оригиналы которых были удалены из медиа-библиотеки.
- **Виджет консоли**: Сводная статистика оптимизации (количество картинок, процент сжатия, объем сэкономленного места) прямо на главной странице админки.
- **Удобный интерфейс**: Отдельная страница управления (CMP) для запуска сканирования, сжатия и очистки файлов.

## Начало работы

### Требования

- Установленная CMS MODX 3.x.
- Утилита `cwebp`, установленная на сервере и доступная для вызова из PHP.

### Установка

1. Скачайте транспортный пакет: [webpconverter-1.0.0-pl.transport.zip](../webpconverter-1.0.0-pl.transport.zip).
2. Загрузите архив через **Менеджер пакетов** (Приложения -> Управление пакетами) в панели управления MODX.
3. Нажмите кнопку **Установить** и следуйте инструкциям на экране.

## Сборка и деплой

Для сборки изменений в новый транспортный пакет выполните скрипт:

```bash
php core/components/webpconverter/_build/build.transport.php
```

Сборочный скрипт автоматически упакует плагины, лексиконы, системные настройки, виджеты и меню в новый архив в папке `core/packages/`. Скопируйте собранный ZIP-архив обратно в корень этой директории для обновления дистрибутива.
